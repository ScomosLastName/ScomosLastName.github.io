Point-Plotting

Category Desktop App
Year 2024
Stack Java (Swing)
Status In Development

Overview

Point-Plotting is a desktop application built in Java, intended for users who need to quickly create and visualize line graphs based on simple point input. This project was inspired by the need to move beyond plotting data in spreadsheets (like Excel or Google Sheets) and instead provide a tailored, user-friendly graphing environment.

The application is a work in progress and is based on a prior project, now reimagined and refactored for better usability and extensibility. It's primarily designed for students, teachers, or anyone who wants an easy way to enter X/Y values and see them visualized as lines or points. It also serves as a way for me to deepen my understanding of Java Swing and desktop UI design.

How it works

Point-Plotting centers around a basic UI using Java Swing. Users can input pairs of X and Y coordinates, add them to a set, and visualize the connections as a line graph. The graph updates dynamically as new points are added or removed.

The main challenges addressed were:

  • Implementing smooth coordinate-to-pixel mapping regardless of window size;
  • Allowing flexible point input and easy clearing or resetting;
  • Drawing scalable axes, gridlines, and customizable point/line styling.

The project is pure Java — no dependencies other than standard JDK. The code is structured for easy future expansion, such as supporting multiple data sets, exporting graphs as images, or enabling more advanced chart types.

What I learned

This project has deepened my experience with Java's Swing framework, particularly dynamic UI updates, custom component painting, and event-driven programming. I learned a lot about mapping between logical (data) coordinates and screen pixels, and about making UI logic extensible for future features. If I were to start again, I would plan earlier for custom axes or zoom/pan interactions, and I would modularize drawing code for even easier extensibility.

Point-Plotting has also helped me develop an eye for user-centric design: simple forms, immediate feedback, and a clear focus on the user's workflow.
